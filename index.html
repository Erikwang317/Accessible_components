Reflection on Kat Holmes’ Inclusive Design

  • The UI components provided feedback through color changes, cursor flashing, and button states, making them easy to understand.
  • From the video, I understand that balancing efficiency and accessibility is essential. While adding an extra step might slow down power users, it makes the interface more inclusive for those who need structured interactions.

Mouse Users:

  • Thinking of people who might have difficulty seeing & hearing, I added a clear color change and screen reader explanations for "Hover" to improve accessibility of the UI.

Keyboard Users:

  • For those unfamiliar with technology, I added an explicit "Enter" confirmation before typing to prevent accidental text input.
  • This benefits functionality users but sacrifices efficiency.

Example 1 (Accessibility) - ChatGPT's Keyboard Accessibility:

  • ChatGPT focuses more on web features compared to mobile.
  • Allows Tab navigation to move through buttons logically.
  • Users with motor impairments or no mouse access can still use all functions efficiently.

Example 2 (Inaccessibility) - Instagram's Search Box:

  • Instagram prioritizes mobile features over web accessibility.
  • The search box on the web version is hidden behind a button that must be tapped first.
  • Users with visual impairments or screen reader reliance may struggle to find the input field quickly.
